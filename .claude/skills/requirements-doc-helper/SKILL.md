# 需求文档助手（前端版）

## 概述

当用户有一个简单的功能想法或业务需求，需要转化为前端开发文档时，使用此Skill。AI会主动与用户确认需求细节，然后**强制调用 Frontend Design Skill** 生成完整的前端界面和组件。

## 适用场景

- 用户说"帮我做一个xxx页面"或"帮我写一个xxx功能的文档"
- 用户需要前端UI设计、组件设计或页面原型
- 用户需要一个完整的前端功能开发方案

## 工作流程

### 第一阶段：需求收集

仔细阅读用户的初始prompt，识别：
- 核心功能目标
- 涉及的页面/组件
- 明显的缺失信息

如果初始需求不完整，使用 `AskUserQuestion` 工具向用户提问。

### 第二阶段：需求确认

从以下维度系统地向用户确认需求：

1. **功能范围**：这个功能需要哪些页面？页面之间的跳转关系是什么？
2. **用户角色**：谁会使用？不同角色的界面是否不同？
3. **技术栈**：React/Vue/其他框架？UI库选择？
4. **交互需求**：需要哪些用户操作？表单校验规则？加载状态？
5. **响应式设计**：是否需要适配移动端？
6. **视觉风格**：品牌色？深色模式？设计风格偏好？

### 第三阶段：生成文档 + 调用Frontend Design

完成需求确认后：

1. **编写开发文档**（包含前端相关内容）
2. **立即调用 Frontend Design Skill**，使用 `Skill` 工具生成完整的前端界面

## 前端开发文档结构

```markdown
# [功能名称] 前端开发文档

## 1. 文档概述
- 背景与目的
- 涉及页面/组件

## 2. 功能需求
- 功能描述
- 用户故事
- 页面清单

## 3. 交互设计
- 用户操作流程
- 状态管理（加载/成功/错误/空状态）
- 表单校验规则

## 4. 组件设计
- 核心组件列表
- 组件API设计
- Props/Events

## 5. 视觉规范
- 颜色/字体
- 间距/布局
- 响应式规则

## 6. 接口对接
- API列表
- 前后端数据约定

## 7. 验收标准
- 功能验收点
- UI验收点
- 兼容性要求
```

## 强制调用 Frontend Design

**重要**：在完成需求文档编写后，必须立即调用 Frontend Design Skill 生成前端代码。

使用方式：
```
Skill 工具 with skill: "frontend-design" and args: "<简要描述需求>"
```

Frontend Design 会生成：
- 完整的 React/Vue 组件代码
- 样式实现
- 交互逻辑

## 输出规范

- 使用中文编写
- Markdown格式，层级清晰
- 文档应完整、可操作
- Frontend Design 生成的代码应直接可用

## 示例

**用户输入**：
> 帮我做一个用户登录页面

**AI追问**：
> 需要确认几个细节：
> - 登录方式：账号密码？手机验证码？第三方登录？
> - 是否需要注册入口？
> - 是否需要记住密码功能？
> - 需要哪些表单校验？
> - 设计风格偏好？

**用户回答后**：
1. AI生成完整的登录页面开发文档
2. AI调用 Frontend Design Skill 生成登录页面代码

## 重要原则

- 不要假设任何需求，必须向用户确认
- 必须调用 Frontend Design Skill 生成实际代码
- 保持文档的实用性和可操作性
